# 이 블로그에 대해

Hugo로 만든 애드센스 수익화 목적 블로그. 주제는 법률(임대차/전세사기 등), 세금, 재테크처럼
CPC가 높은 카테고리를 다룬다. 콘텐츠는 정보성이되, 실행형/비교형 키워드(신청 방법, 비교,
조건 등)를 우선한다.

## 새 글 작성 시 반드시 지킬 구조

### Front matter

```yaml
---
title: "핵심 키워드를 앞에 둔 제목"
date: YYYY-MM-DD
draft: true   # 검수 끝나면 false로 변경 후 발행
categories: ["대분류"]   # 사이트 상단 카테고리 클러스터 (예: 법률, 세금, 재테크)
tags: ["롱테일 키워드1", "롱테일 키워드2"]   # 본문에서 실제로 다룬 구체적 롱테일
description: "검색결과 미리보기 문장. 클릭하면 얻는 이득을 압축해서 쓴다."
---
```

### 본문

1. **서론은 두괄식**: 첫 문단에서 바로 답한다. "~에 대해 알아보겠습니다" 같은 예고형 서론 금지.
2. **소제목은 구체적으로**: "장점"이 아니라 "OO인 경우 손해 보는 이유"처럼 스캔하다 멈출 만큼
   구체적으로 쓴다. 각 소제목 아래엔 수치·단계·조건 등 실질 정보를 담는다.
3. **공식 신청/신청 방법 관련 글이면 실제 신청 링크를 반드시 포함**한다.
4. **FAQ 3~5개를 글 끝에 배치**한다. 본문에서 다룬 내용을 기반으로 하되, 본문 제목과는 다른
   각도의 진짜 롱테일 질문으로 구성한다 (본문이 다루지 않은 인접 질문을 흡수하는 그물 역할).
5. **YMYL(법률/세금/의료/금융) 주제는 팩트체크 필수**. 확실하지 않은 수치·조건은 웹 검색으로
   최신 정보를 확인하고, 글 하단에 "정보 제공 목적이며 정확한 확인은 공식 기관에서" 식의
   면책 문구를 넣는다.

### 카테고리/태그 체계

- `categories`: 큰 주제 클러스터 (사이트 상단 "카테고리" 메뉴에서 모아보기, `/categories/`)
- `tags`: 세부 롱테일 키워드 (`/tags/`)
